## Bugs fixed in 2026-06-25 QA pass
A full click-through QA pass (homepage, TyreFinder, shop retail/dealer, cart, checkout, responsive breakpoints) found and fixed, all verified locally and with a clean `npm run build`:
- **Size-search deep links were silently broken site-wide** — biggest find. `components/shop/RetailView.tsx` called `useProducts()` with no arguments, so the `width`/`profile`/`rimSize` query params generated by the homepage's "ดูสินค้า" cards and the TyreFinder's main search button (e.g. `/shop?mode=customer&width=205&profile=55&rimSize=16`) were completely ignored — every one of those links landed on the full unfiltered 489-product catalog instead of the matching size. Fixed by reading the params via `useSearchParams()` and passing them into `useProducts()`, which already had working filter support and an already-correct `/api/products` route — the gap was purely that nothing wired the URL into it. Verified: that exact URL now correctly shows "14 Tyres Available", all 205/55R16. Dealer mode intentionally untouched — it's a full-catalog bulk-ordering view by design and nothing links to it with size params.
- **Mobile nav overlay could block the entire desktop site.** `components/shared/Navbar.tsx`'s mobile menu overlay (`fixed inset-0 z-50`) had no `md:hidden`, unlike its trigger button — so if a user opened the mobile menu then widened the window (resize, rotate, unfold), the full-screen overlay stayed up and blocked all clicks at desktop width too. Added `md:hidden` to match the trigger.
- **React console warning** ("Select is changing from uncontrolled to controlled") on all 7 `<Select>` instances in `components/tyre-finder/TyreFinder.tsx` — they used `value={x ? String(x) : undefined}`, toggling the prop between a string and `undefined` across renders. Changed the unset case to `''` (the Radix-recommended sentinel) on all 7. Note: one instance of this same warning text persists after the fix — confirmed to be an unrelated known upstream `@radix-ui/react-select@2.1.0` + React 19 compatibility quirk in Radix's internal hidden-native-select mirror, not fixable from app code; cosmetic, dev-console-only, no functional impact.
- Minor: added `data-scroll-behavior="smooth"` to `<html>` in `app/layout.tsx` to silence a Next.js advisory about `scroll-behavior: smooth` during route transitions.
- **Noted but not changed (content, not code, needs your decision):** the LINE links across the whole site (navbar, footer, hero, LINE floating button) all point to `https://line.me/ti/p/@tirehub` — the pre-rebrand handle. If your actual LINE OA account is still under that name this is correct; if you've since renamed it, every "Message us on LINE" button site-wide is pointing at the wrong place. Also, the footer phone link is `tel:+6620000000`, which looks like a placeholder, not a real number — worth checking.

## Tech Stack
- **Framework:** Next.js 16 (App Router), React 19, TypeScript
- **Styling:** Tailwind CSS v4 (CSS-first `@theme` tokens in `globals.css`)
- **Animation:** Framer Motion
- **State:** Zustand v5 (`useShopStore`, `useFilterStore`)

### 3. User Authentication — built, then deliberately removed (2026-06-23)
Auth (sign in/up, role-gated dealer mode) was built earlier this session, then the owner reversed that decision: **this site is a product catalog/showcase that drives customers to LINE OA for the actual sale — not a transactional app, so no login/account system at all.** Forcing signup before browsing Dealer mode was bad UX for that model. Removed entirely:
- Deleted `app/auth/` (page + callback route), `components/shared/AuthControl.tsx`, `components/shop/DealerAccessGate.tsx`, `context/SupabaseSessionProvider.tsx`, `lib/supabase/client.ts` + `server.ts`, and `proxy.ts` (middleware whose only job was refreshing a Supabase session that no longer exists).
- `components/shop/ShopPageClient.tsx`: Dealer mode now renders `DealerMatrix` directly, same as Retail — no redirect, no gate.
- `app/api/products/route.ts`: no more dealer/customer branching — always returns full data (`pricingTiers`, `inventory`/warehouse stock) to everyone, since there's no way to tell visitors apart anymore.
- **Consequence to remember**: once real ERP prices load (TO-DO #1), the Dealer view's volume-pricing tiers will be publicly visible to any visitor, not just "dealers" — there's no access control left to bring back without rebuilding auth. Fine for now since those numbers are all ฿0 placeholders.
- Backend Supabase Auth tables/RLS (`profiles`, `is_dealer()`, the `orders_select_own` policy, etc.) were deliberately left untouched in the database — removing schema is riskier than removing frontend code, and it costs nothing to leave unused. `/api/orders` now always creates guest orders (no `userId` lookup).
- Build output changed nicely as a side effect: `/`, `/shop`, `/checkout` are now static pages (○) instead of dynamic (ƒ), since nothing depends on cookies/session at request time anymore.

The upload script (`scripts/upload-product-images.mjs`) is unchanged and ready the moment `BRAND__MODEL.jpg` files exist.

- **2026-06-24: `scripts/upload-product-images.mjs` written and tested** — drop files named `BRAND__MODEL.jpg` (e.g. `DUNLOP__SP_LM705.jpg`) into a folder, run `node --env-file=.env.local scripts/upload-product-images.mjs <folder>`, it uploads to the bucket and sets `image_url` on every matching product (one tread photo covers all size variants of that brand+model). Tested against an empty folder, runs clean.

Decided not to scrape any brand site without an explicit license.
